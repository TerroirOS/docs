    ფრონტენდის არქიტექტურა¶

    დღეს repo-ის frontend არის public website და docs; ოპერაციული frontend მომავალ reference layer-ად უნდა წავიკითხოთ.

    reference frontend design

    producer/issuer/public operational frontend ამ რეპოზიტორიაში ჯერ არ არის; დღეს გვაქვს docs და marketing experience.

    frontend-ის საზღვარი¶

    მომავალ frontend-ს უნდა შეეძლოს producers, issuers და public verifiers სხვადასხვა trust state-ების მკაფიოდ დანახვა. დღევანდელი frontend კი public site-სა და documentation layer-ს მოიცავს.
