# Who Terroir serves

The documentation is designed so different audiences can enter through their own questions, whether they need a practical workflow, a verification checklist, or an architectural explanation.

!!! tip "Audience map"
    This page maps the current documentation experience and recommended reading order for each audience.

## Four primary audiences

### Choose the shortest useful route

| Audience | Main question | Recommended first pages |
| --- | --- | --- |
| General reader | What is Terroir and why does it exist? | What is Terroir, Why trusted origin matters, FAQ |
| Producer or certifier | How would this support a real-world origin workflow? | Producer journey, Certifier onboarding, Exporter workflows |
| Buyer or importer | How do I decide whether to trust a record? | Buyer and importer guide, How verification works, Example product passport |
| Developer or technical reviewer | What is implemented, what is planned, and what are the interfaces? | Architecture, Implementation status, Technical reference pages |

!!! info "English-first rollout"
    The expanded library is shipping in English first. Existing Georgian pages remain available, while new pages fall back to English when opened directly under the Georgian locale.

## How to get value from the docs quickly

### Documentation reading pattern

1. **Start with the plain-language summary** - Every page opens with a non-technical explanation so you can decide whether the page is relevant.

2. **Check the status label** - Status notes tell you whether the page describes the website as it exists today, a current content artifact, or a planned interface.

3. **Use related pages** - Each page ends with next-step links so you do not need to guess the right reading sequence.

4. **Use citations when needed** - Research-backed claims include inline source markers that jump to references at the bottom of the page.
