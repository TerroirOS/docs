# Producer journey

The producer journey explains how a future Terroir workflow should move from batch creation to outside verification without forcing the producer to become a technical expert.

!!! warning "Reference workflow"
    This is a future-facing workflow guide. The current repository does not yet implement a live producer dashboard or batch-management application.

## Producer goal

A producer should be able to describe a batch once, attach the right evidence over time, and share a verification story without needing to master the underlying infrastructure. The product experience should reduce paperwork friction, not replace one confusing process with another.

## Reference producer workflow

### Producer path

1. **Create the batch record** - Define the product lot, origin context, and initial narrative in a stable passport.

2. **Attach supporting evidence** - Link the operational records that support the origin story or later certification needs.

3. **Invite or coordinate attestations** - Share the batch with certifiers, labs, cooperatives, or other trusted actors.

4. **Review verification readiness** - See whether the batch is complete enough for export, compliance, or consumer presentation.

5. **Publish or share the verification view** - Create the readable output used by buyers, importers, or consumers.

## What producers need from the product

### Non-negotiables

- Clear separation between draft work and externally visible trust signals.

- The ability to reuse data across export, certification, and storytelling workflows.

- Simple language explaining which missing items actually block verification.

- Visible history when important fields change or claims are superseded.
