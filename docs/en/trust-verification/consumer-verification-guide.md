# Consumer verification guide

This guide explains how an everyday reader should review a product passport or verification page without needing to understand the underlying technical stack.

!!! info "Public-facing guide"
    This guide is written for a future public verification experience. It is still useful now as a content and UX benchmark.

## Five things to check

### Public verification checklist

1. **Read the summary first** - Make sure the product, producer, and region are easy to understand in plain language.

2. **Look for who verified what** - Do not rely on a generic badge alone; check whether the page names the organization behind key claims.

3. **Check the status** - A trustworthy page tells you whether the record is verified, incomplete, changed later, or under review.

4. **Open supporting evidence only if needed** - If something matters to your decision, confirm that the supporting document or note exists.

5. **Notice what is missing** - Absence of evidence is not the same as a failed claim, but it should lower confidence when the claim matters.

## Language that builds trust

### Copy choices matter

| Better wording | Avoid |
| --- | --- |
| Verified against the currently committed record | Guaranteed authentic forever |
| No matching attestation is currently available | Everything is fine |
| This record changed after an earlier commitment | Minor update with no explanation |
| Issued by the named organization | Certified, without naming by whom |
