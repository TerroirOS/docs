# Contribution process

The contribution process should help people improve the project without accidentally widening the gap between documentation and implementation reality.

!!! tip "Current process guidance"
    This page reflects how contributors should approach changes in the current repository, especially for trust-sensitive documentation.

## Contribution principles

### Rules that matter most

- Do not let the docs imply live functionality that the repo does not contain.

- Prefer primary or authoritative sources when adding externally grounded claims.

- Use plain-language summaries before technical depth so the docs serve mixed audiences.

- Preserve bilingual stability for the translated pages that already exist.

## Suggested contribution flow

### Documentation contribution flow

1. **Confirm the current repo state** - Inspect the code before editing docs that describe implementation.

2. **Gather sources** - Use authoritative sources and add inline citations when the page makes factual external claims.

3. **Update content and navigation together** - When a page moves or expands, keep the sidebar and related pages aligned.

4. **Lint and review trust language** - Treat wording around verification, guarantees, and implementation status as a review focus, not just grammar.
