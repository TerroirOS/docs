# FAQ

The FAQ answers the questions people tend to ask when they are still deciding whether Terroir is a website, a concept, a future product, or an open infrastructure effort.

!!! tip "Public explainer"
    This page is part of the current documentation experience and intentionally mixes plain-language answers with candid implementation context.

## Frequently asked questions

#### Is Terroir already a full working provenance platform?

Not in this repository. What exists today is the public website, documentation, research library, and content framing for the broader platform direction.

#### Why does the documentation still describe future APIs and contracts?

Because those interfaces are still useful as reference architecture and planning material, but the rewritten docs now mark them clearly as planned or reference-only.

#### Why start with Georgian wine?

Because origin is central to its value, the cultural context is unusually strong, and the use case makes the trust problem easy to understand for both technical and non-technical readers.

#### Can these ideas apply beyond wine?

Yes. The goal is a reusable origin-and-attestation pattern that can later support other crops and regions without treating Georgian wine as the only valid use case.
