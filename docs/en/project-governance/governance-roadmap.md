# Governance roadmap

The governance roadmap explains how Terroir could move from a team-led public project into a more durable community-governed trust infrastructure.

!!! info "Public roadmap"
    This page documents intended governance direction alongside the current documentation and website implementation.

## Why governance matters here

Provenance infrastructure only becomes trustworthy if the rules around issuers, schemas, and verification do not quietly collapse into private control. Governance is therefore part of the product, not a later administrative extra.

The digital public goods lens is useful here because it treats openness, documentation, and community accountability as part of long-term infrastructure quality rather than just branding.[1]

## Roadmap phases

### Governance progression

| Phase | Primary focus | What success looks like |
| --- | --- | --- |
| Foundation | Clear docs, transparent architecture, and a coherent pilot story. | People can understand the system boundary and contribute without guesswork. |
| Operational pilots | Limited live workflows with explicit governance roles. | A small set of trusted actors can run the system without hiding decisions. |
| Shared stewardship | Broader schema and issuer governance with documented review paths. | Changes become discussable and reversible by the community, not just a single team. |
