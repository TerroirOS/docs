# Licensing

Licensing tells collaborators how they may reuse the code and content. Terroir documentation should make this explicit because openness is part of the trust story.

!!! tip "Current licensing guidance"
    This page documents the open-source licensing references used in the project narrative and explains how to think about reuse.

## Plain-language explanation

An open-source license answers a practical question: what can another person or organization legally do with the code? Terroir documentation should make that clear up front because trust infrastructure loses credibility if reuse rights are vague.

## Core references

### Licensing building blocks

| Reference | What it covers | Why it matters here |
| --- | --- | --- |
| Apache License 2.0 | Reuse, modification, distribution, and patent grant terms. | Sets expectations for open reuse where that license is adopted.[1] |
| Developer Certificate of Origin | Contributor sign-off language for submitted work. | Keeps contribution provenance explicit without requiring a separate CLA in every case.[2] |
