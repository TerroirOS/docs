# Glossary

The glossary defines the terms that appear repeatedly across the docs so readers from different backgrounds can keep a shared mental model.

!!! tip "Current reference aid"
    This glossary supports the current documentation set and will likely expand as implementation work adds more concrete interfaces.

## Key terms

- **Attestation:** A structured claim made by a specific actor about a batch, process, or event.

- **Batch / passport:** The stable record describing a concrete product lot and the trust story that accumulates around it.

- **Integrity commitment:** A hash or similar commitment that helps later readers detect silent change.

- **Issuer:** An actor whose role allows them to make a certain class of attestation.

- **Verification view:** The readable surface that packages the underlying provenance record for review by another person.

- **Superseded:** A state indicating that a record or claim has been replaced by a newer authoritative version rather than silently edited.
