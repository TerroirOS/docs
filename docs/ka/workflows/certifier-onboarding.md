# სერტიფიკატორის ონბორდინგი

Certifier onboarding არის governance ნაბიჯი, სადაც იკვეთება ვინ რა ტიპის claim-ს გასცემს.

!!! warning "reference issuer workflow"
    სერტიფიკატორის registry და approval flow ამ ეტაპზე ჯერ არ არის განხორციელებული.

## რატომ არის ეს ფრთხილი პროცესი

თუ authority ყველასთვის ღია და გაუმჭვირვალეა, provenance სისტემის ნდობა სწრაფად ინგრევა. ამიტომ certifier onboarding უნდა იყოს კონტროლირებული governance პროცესი და არა უბრალო თვითრეგისტრაცია.

პრაქტიკულად ეს ნიშნავს ორგანიზაციული იდენტობის დადასტურებას, claim scope-ის განსაზღვრას და review/renewal წესების დოკუმენტირებას.
