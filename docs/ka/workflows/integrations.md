# ინტეგრაციის გზამკვლევები

Terroir-ის იდეა არსებულ სამუშაო პროცესებთან დაკავშირებაა და არა ყველა სისტემის ერთდროული ჩანაცვლება.

!!! warning "reference integration patterns"
    ინტეგრაციის ეს აღწერა სამომავლოა; შესაბამისი API-ები და სერვისები ამ რეპოზიტორიაში არ არის.

## ინტეგრაციის ძირითადი იდეები

სამომავლო ინტეგრაციები შეიძლება ეხებოდეს producer-სისტემების data sync-ს, issuer workflow-ების დაკავშირებას ან verification view-ის ჩაშენებას სხვა public თუ trade ინტერფეისებში.

მთავარი მიზანია provenance ჩანაწერი პორტაბელური იყოს და adoption ეტაპობრივად მოხდეს, ისე რომ მთელი ჯაჭვის ერთდროული გადაყვანა აუცილებელი არ გახდეს.
